# Security Policy

## Reporting a vulnerability
If you discover a security issue, please do not open a public issue with sensitive details.
Instead, contact the repository owner via GitHub Security Advisories (preferred) or direct message.

## What not to include in this repository
- Secrets (tokens, keys, passwords, certificates)
- Real customer data, personal data (PII/PHI), or confidential internal documents
- Any sensitive logs or exports that could identify individuals or systems

## Design principles
- Least privilege by default
- Human-in-the-loop approval for risky actions
- Synthetic examples only unless explicitly approved and handled safely
